## Session 2026-07-08-03:00 — Continuous build (feature/mvp-end-to-end-demo)

**Scope:** Maintainer opted to drop the per-role branch/STATUS ceremony for
this local research prototype and move fast toward a working end-to-end
demo, with the backend explicitly stubbed (hardcoded "cleared" + hardcoded
tags, no real Chromaprint/librosa/PANNs/Qdrant — that's a separate, later

**Changed:**
- `extension/src/capture/capture-engine.ts` — `startCapture()`: resolves the
  active adapter, wires `AudioContext` → adapter's `getAudioNode()` →
  `MediaStreamAudioDestinationNode` → `MediaRecorder`, also reconnects to
  `audioContext.destination` so page audio isn't muted while capturing.
- `extension/src/content-script.ts` — mounts a floating capture button
  (only when `getActiveAdapter()` finds a match), drives a small
  idle→recording→uploading→done state machine, `fetch`s the stub backend.
- `extension/manifest.json` — MV3, `<all_urls>` content script + host
  permissions (capture needs to work on any adapter-matched page).
- `extension/package.json` — added `esbuild` (dev-only) + `build` script
  bundling `content-script.ts` into `dist/content-script.js` (IIFE; browsers
  can't run the raw ESM `import`/`export` content scripts otherwise).
- New `backend/` — stub Express server (`server.js`): `POST /upload`
  (multer, hardcoded copyright/tag response) and `GET /download/:id`
  (in-memory `Map`, resets on restart — fine for a local stub, not a DB).
- **Fixed a pre-existing bug in the repo's own `.gitignore`:** every pattern
  had a trailing comma (`node_modules/, `), which made them literal strings
  that matched nothing — `node_modules/`, `dist/` were never actually being
  ignored. Verified with `git check-ignore` before and after. Added
  `uploads/` for the new backend stub's upload directory.

**Broke / flaky:** Nothing broke. Independently verified, not just typed and
assumed:
- `npm run typecheck` (extension) — clean.
- `npm run build` (extension) — bundles clean, 11.4kb, syntax-checked with
  `node --check`.
- `npm test` (extension) — 5/5 passing (unchanged from task-001).
